## はじめに

このページでは、Claude Code Workflow Studioについてよく寄せられる質問とその回答をまとめています。お探しの情報が見つからない場合は、GitHubのIssuesでお問い合わせください。

## 一般的な質問

### Q1: Claude Code Workflow Studioとは何ですか？

**A:** Claude Code Workflow StudioはVSCode拡張機能で、Claude Codeのワークフローを視覚的に設計できるツールです。ノーコードでドラッグ&ドロップ操作により、Sub-Agentの組み合わせや条件分岐を含む複雑なワークフローを作成できます。

設計したワークフローは`.claude`ファイルとしてエクスポートでき、Claude Codeですぐに実行可能です。

### Q2: Claude Codeとは何が違いますか？

**A:** 役割が異なります：

- **Claude Code**: ワークフローを**実行**するCLIツール
- **Claude Code Workflow Studio**: ワークフローを**設計**するVSCode拡張機能

Workflow Studioで設計したワークフローをClaude Codeで実行する、という関係です。

### Q3: 無料で使えますか？

**A:** 拡張機能自体は無料です。ただし、エクスポートしたワークフローをClaude Codeで実行する際には、Claude APIの利用料金が発生します（Anthropicの価格体系に従います）。

### Q4: プログラミング経験は必要ですか？

**A:** いいえ、必要ありません！ビジュアルエディタでドラッグ&ドロップ操作だけでワークフローを作成できます。マークダウンの構文やプログラミングの知識は不要です。

ただし、Sub-Agentのプロンプトを記述する際には、タスクを明確に説明する能力が役立ちます。

## インストールと設定

### Q5: インストール方法を教えてください

**A:** VSCode Marketplace から簡単にインストールできます（Coming Soon）：

1. VSCodeを開く
2. 拡張機能ビュー（`Ctrl+Shift+X` / `Cmd+Shift+X`）
3. "Claude Code Workflow Studio" を検索
4. **Install** をクリック

現在はMarketplace公開準備中です。公開され次第、こちらで案内します。

### Q6: Claude Codeは必須ですか？

**A:** はい、エクスポートしたワークフローを実行するにはClaude Codeが必要です。Claude Codeのインストール方法は、[公式ドキュメント](https://docs.anthropic.com/claude/docs/claude-code)をご覧ください。

ただし、ワークフローの設計だけであれば、Claude Codeなしでも可能です（エクスポートしたファイルを後で使う場合）。

### Q7: オフラインで使えますか？

**A:** はい！Claude Code Workflow Studioは完全オフラインで動作します。ネットワーク通信は一切行わないため、プライバシーとセキュリティが保護されます。

ワークフローの設計、保存、エクスポートすべてがローカルで完結します。

## 使い方

### Q8: ワークフローはどこに保存されますか？

**A:** 2つの保存先があります：

1. **設計データ**: `.vscode/workflows/*.json`
   - ビジュアルエディタで再編集可能な形式
   - Gitで管理してチームと共有可能

2. **エクスポートファイル**: `.claude/agents/` と `.claude/commands/`
   - Claude Codeで実行可能な形式
   - Markdownフロントマター形式

### Q9: ワークフローを他の人と共有できますか？

**A:** はい、簡単に共有できます：

```bash
# 設計データをコミット
git add .vscode/workflows/my-workflow.json
git commit -m "Add my workflow"
git push
```

チームメンバーがpullすれば、同じワークフローをビジュアルエディタで開いて編集できます。

### Q10: ノードは何個まで追加できますか？

**A:** 1つのワークフローにつき最大50ノードまでです。ほとんどのワークフローは3〜10ノードで設計できます。

より複雑な処理が必要な場合は、ワークフローを複数に分割し、それぞれをSlashCommandとして実行することを推奨します。

### Q11: ワークフローの実行順序はどう決まりますか？

**A:** ノード間の**接続の順番**で決まります。出力ポート（右側）から入力ポート（左側）へ接続することで、実行順序が定義されます。

AskUserQuestionノードでは、ユーザーが選択した選択肢に応じて分岐します。

## トラブルシューティング

### Q12: エディタが開きません

**解決方法**：

1. VSCodeを再起動
2. 拡張機能が正しくインストールされているか確認
3. VSCodeのバージョンを最新に更新
4. 開発者ツール（`Ctrl+Shift+I` / `Cmd+Option+I`）でエラーログを確認

### Q13: ワークフローが保存できません

**原因と解決方法**：

**原因1**: ワークフロー名に使用できない文字が含まれている
- 英数字、ハイフン、アンダースコアのみ使用してください
- スペースや特殊文字は使えません

**原因2**: 必須フィールドが未入力
- すべてのノードに名前を設定してください
- Sub-AgentノードにはPromptが必須です

**原因3**: `.vscode/workflows/` ディレクトリがない
- 自動的に作成されるはずですが、手動で作成してみてください

### Q14: エクスポートしたファイルが動きません

**確認事項**：

1. **Claude Codeの再起動**: 新しいコマンドを認識させるため
2. **ファイルの場所**: `.claude/commands/` に正しく配置されているか
3. **ファイル形式**: Markdownフロントマターが正しいか
4. **エージェントファイル**: `.claude/agents/` に必要なエージェントがあるか

### Q15: ワークフローリストが更新されません

**解決方法**：

1. Load ドロップダウンの更新ボタン（↻）をクリック
2. エディタを閉じて再度開く
3. VSCodeを再起動

### Q16: ノードをうまく接続できません

**コツ**：

- 出力ポート（ノードの右側）から開始
- 入力ポート（ノードの左側）までドラッグ
- ポートにマウスを合わせるとハイライトされます
- Ctrlキーを押しながらだと直線で接続されます

## 高度な使い方

### Q17: 変数やパラメータは使えますか？

**A:** はい、Promptノードで`{{variableName}}`構文を使用できます：

```
解析対象: {{fileName}}
出力形式: {{format}}
```

実行時にClaude Codeが値を代入します。

### Q18: 条件分岐はどうやって作りますか？

**A:** 2つの方法があります：

1. **AskUserQuestion ノード**: ユーザーが選択肢を選ぶ
2. **Branch ノード**: 条件式で自動判定

複雑な条件は、Sub-Agent内のプロンプトで記述することも可能です。

### Q19: エラーハンドリングはできますか？

**A:** Sub-Agentのプロンプト内で指示できます：

```
データファイルを読み込んでください。
ファイルが見つからない場合は、ユーザーに報告して処理を中断してください。
```

また、Branchノードで「ファイル存在チェック」などの条件を設定することも可能です。

### Q20: 多言語対応していますか？

**A:** はい！VSCodeの表示言語設定（`vscode.env.language`）に自動対応します：

- **UI**: ツールバー、ノードパレット、プロパティパネル
- **エクスポート**: 生成される`.claude`ファイルのドキュメント

対応言語：英語、日本語、韓国語、簡体字中国語、繁体字中国語
